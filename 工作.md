## 后端

### JWT鉴权，登录

登录时，服务器根据：

```python
{
	username: "xxxxxx",
    exp: "xxxx-xx-xx"
    # 过期时间
}
```

通过secret，私钥

生成token并返回给客户端，客户端存在本地

客户端请求时必须带上：

```python
{
	”Authoration“: Bearer {{token}}
}
```

e.g. http://localhost:8000/login

### Topic订阅

关于Topic订阅，出现一些问题

它流程是这样，订阅Topic之后，broker会向客户端不定时推送有关这个topic的内容

这和传统的请求-响应模式有所不同

需要客户端建立到服务器端的稳定双向连接，当服务器端接收到来自broker的topic推送后，由服务器端主动将推送发送到前端，以此实时显示。

### 图片导入

静态引入，比如头像固定不变

```vue
<script setup>
import avatar from '@/assets/avatar1.png'
</script>

<template>
  <img :src="avatar" />
</template>
```

动态导入

```vue
<script setup>
import avatar1 from '@/assets/avatar1.png'
import avatar2 from '@/assets/avatar2.png'
import avatar3 from '@/assets/avatar3.png'

const avatars = [avatar1, avatar2, avatar3]

const current = ref(0)

function changeAvatar() {
  current.value = (current.value + 1) % avatars.length
}
</script>

<template>
  <img :src="avatars[current]" />
  <button @click="changeAvatar">换头像</button>
</template>
```

如果在public目录

```vue
<img :src="`/avatars/${name}.png`" />
```

批量导入

```vue
const modules = import.meta.glob('@/assets/avatars/*.png', { eager: true })

const avatars = Object.values(modules).map(m => m.default)
```
